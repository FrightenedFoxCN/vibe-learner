# Plan Text Contract

This document defines the learner-facing text fields used by learning plans across backend models, shared contracts, prompts, and frontend UI.

## Why This Exists

The repository uses one persisted `LearningPlan` shape across Python and TypeScript, but different fields serve different UX layers:

- some text is title-level and stable
- some text is summary-level and generated
- some text is compact label text
- some text is actionable task text

These fields must not be used interchangeably.

## Canonical Field Meanings

### `course_title`

- Source of truth: generated by heuristic planner or model planner
- Ownership: persisted on `LearningPlan`
- Meaning: textbook-grounded course title derived from cleaned study units
- UI role: main heading of the `Plan Overview Panel`
- Writing rule: concise course or chapter title, readable as a heading
- Must not be used as: learner goal statement, summary paragraph, or task text

Examples:

- `高等数学 / 极限与连续`
- `Discrete Mathematics / Chapter 1 Foundations`

### `objective`

- Source of truth: learner input at plan creation time
- Ownership: request field on `LearningGoalInput`, persisted on `LearningPlan`
- Meaning: the learner's study goal
- UI role: supporting metadata text below or beside the main title
- Writing rule: can be sentence-like, but should still read as a goal statement
- Must not be used as: generated summary, chip text, or task list item

Examples:

- `在两周内掌握这本教材的第一章并完成一次复习。`
- `Prepare for the linear algebra midterm by finishing Chapters 1-3.`

### `overview`

- Source of truth: generated by heuristic planner or model planner
- Ownership: response field on `LearningPlan`
- Meaning: short learner-facing summary of the plan
- UI role: summary/body text below the title
- Writing rule: 1-2 sentences, readable as a concise paragraph
- Must not be used as: the main heading of the plan panel

Examples:

- `本周先完成向量基础与例题精读，并在两次学习间插入一次短复习。`
- `This first-pass plan covers the foundations chapter, then schedules a short review before practice.`

### `weekly_focus`

- Source of truth: generated by heuristic planner or model planner
- Ownership: response field on `LearningPlan`
- Meaning: ordered topical focus items for the current week
- UI role: vertical ordered study sequence
- Writing rule: short topic lines, not full instructions
- Must not be used as: long explanations or imperative task text

Examples:

- `向量定义`
- `基底与线性组合`
- `Chapter 1 Foundations`

### `today_tasks`

- Source of truth: generated by heuristic planner or model planner
- Ownership: response field on `LearningPlan`
- Meaning: actionable learner tasks for the current session/day
- UI role: vertical ordered action list
- Writing rule: imperative or task-oriented text
- Must not be used as: summary paragraph or topic chips

Examples:

- `阅读第 12-18 页，标出 2 条定义与 1 个例题。`
- `用自己的话复述牛顿第一定律，并记录一个不确定点。`

## Cross-Layer Mapping

### Backend

- `LearningGoalInput.objective`: learner-authored goal
- `LearningPlanRecord.course_title`: generated plan header title
- `LearningPlanRecord.objective`: persisted learner goal text
- `LearningPlanRecord.overview`: generated summary text
- `LearningPlanRecord.weekly_focus`: ordered weekly study topics
- `LearningPlanRecord.today_tasks`: actionable daily tasks

### Shared TypeScript

- `LearningGoal.objective`
- `LearningPlan.courseTitle`
- `LearningPlan.objective`
- `LearningPlan.overview`
- `LearningPlan.weeklyFocus`
- `LearningPlan.todayTasks`

### Frontend Display

- `Plan Overview Panel` title: `courseTitle`
- `Plan Overview Panel` supporting goal text: `objective`
- `Plan Overview Panel` summary text: `overview`
- `Plan Overview Panel` ordered weekly sequence: `weeklyFocus`
- `Plan Overview Panel` ordered task list: `todayTasks`
- `Plan Overview Panel` recent schedule timeline: `schedule`

## Naming Rules For Future Changes

When adding new plan text fields, choose names by UI role:

- use `title` only for explicit heading-level text
- use `summary` or `overview` only for paragraph-level summaries
- use `focus` for compact topical labels
- use `task` for actionable learner steps

Do not overload one field so that it must behave as both title text and summary text.
