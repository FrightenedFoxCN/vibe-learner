# 氛围学习（vibe learning）用户手册

本文面向产品使用者与测试，目标是把前端每个页面、每个主要部件讲清楚：它是什么、做什么、怎么用、产出什么。

## 1. 使用前准备

推荐先确认本机软件依赖：

1. Git >= 2.40
2. Node.js >= 20
3. npm >= 10
4. Python >= 3.9
5. uv（建议最新稳定版）

安装完依赖后再执行：

1. 启动前端：在仓库根目录执行 npm run dev:web。
2. 启动后端：在 services/ai 目录执行 uv run uvicorn app.main:app --reload。
3. 默认前端会访问 http://127.0.0.1:8000。
4. 若计划使用真实模型，先在 services/ai/.env 配置 litellm 相关变量。

## 1.1 配置数据链路总览

### A. 两条配置来源

1. 启动基线配置：后端读取 services/ai/.env（以及进程环境变量）。
2. 运行时配置：统一设置页通过 API 写入后端本地存储。

### B. 实际生效优先级

1. 后端启动时先读取环境变量，形成 base settings。
2. Runtime Settings 服务会尝试读取本地运行时配置文件。
3. 若已存在运行时配置文件，则以它为主；若不存在，则用环境变量初始化并落盘。

对应本地文件：

- services/ai/data/runtime_settings/default.json

### C. 设置页是如何写入并生效的

1. 统一设置页加载时调用 GET /runtime-settings。
2. 用户修改字段后，前端自动保存触发 PATCH /runtime-settings。
3. 后端写入 services/ai/data/runtime_settings/default.json。
4. 后端容器立即调用 update_runtime_settings，重建 model_provider。
5. 计划生成、人格辅助、章节对话等后续请求立刻使用新配置。

结论：设置页保存后不需要重启后端就会生效。

### D. 环境变量与设置页的职责边界

- 环境变量
  - 更适合首次启动基线、团队默认值、部署时注入。
- 设置页
  - 更适合日常调试、实时切换 provider、模型回填与参数微调。

### E. 关键回退规则（字段级）

- 分场景 API Key（plan/setting/chat）留空 -> 回退全局 openai_api_key。
- 分场景 Base URL（plan/setting/chat）留空 -> 回退全局 openai_base_url。
- 运行 provider 若设为 litellm 但缺少有效 key，后端会回退到 mock provider。

### F. 模型探测与持久化的关系

- 设置页中的 拉取模型与能力 会调用 /runtime-settings/check-openai-models。
- 该探测请求本身不改配置文件，只返回可用模型与能力信息。
- 只有真正修改设置字段并保存（例如模型名、能力开关）才会持久化。
- 若当前是 LiteLLM SDK 直连而上游没有提供 `/models` 兼容接口，则模型探测可能为空，此时直接手填模型名即可。

### G. 如何强制重新以 .env 为准

1. 停掉后端服务。
2. 删除 services/ai/data/runtime_settings/default.json。
3. 重启后端。

重启后会再次用当前环境变量初始化运行时配置。

## 2. 全局导航与通用交互

### 2.1 左侧 TopNav

所有主页面都带有同一套侧边导航（TopNav）。

- 导航链接
  - 导航首页
  - 计划生成

  - 可折叠侧栏，折叠状态会写入本地存储，刷新后保留。

- 加载态：显示 加载中、处理中、拉取中 等文本。
- 失败态：显示错误条或错误文本，不吞掉错误。
- 禁用态：关键操作在忙碌时禁用，避免重复提交。

## 3. 页面与部件详解

## 3.1 导航首页（/）

页面定位：模块入口聚合页。

部件清单：

- 页面头部
  - 标题：学习工作台。
  - 副标题：AI 辅助学习系统的总览定位。
  - 分割线：强化视觉分区。
- 功能入口卡片网格
  - 每张卡包含编号、名称、描述、跳转箭头。
  - 目前包含 6 个主入口：计划生成、章节对话、人格色谱、场景搭建、感官工具、统一设置。

建议操作：首次进入先读卡片描述，按 计划生成 -> 章节对话 的顺序使用。

## 3.2 计划生成（/plan）

页面定位：学习计划与前置配置主工作台。

页面结构：左侧配置列 + 右侧计划列。

### 3.2.1 页面头部

- 页面标题与说明
  - 强调先定人格与场景，再上传教材。
- Notice 提示条
  - 显示计划切换、会话创建、数据刷新等流程反馈。

### 3.2.2 左侧：计划设置区（DocumentSetup）

A. 区块：上传前配置

- 教师人格
  - 使用 PersonaSelector 选择当前计划的人格。
- 计划使用场景
  - 选择场景库中的场景，或回退到本地场景草稿。
- 操作按钮
  - 去场景编辑：跳转到场景搭建页。
  - 创建并打开章节对话/打开章节对话：进入学习会话。

B. 区块：教材上传与分析

- 教材文件输入
  - 仅接收 PDF。
- 学习目标输入
  - 文本域，作为计划生成意图提示。
- 上传并生成计划按钮
  - 触发上传、解析、计划生成主流程。

C. 区块：计划生成轮次

- 轮次总览
  - 展示流式阶段状态、轮次数、工具调用数、恢复次数。
- Round 明细卡
  - 每轮显示状态、finish reason、耗时、错误、恢复说明、工具调用标签。

D. 区块：状态摘要四宫格

- 教材状态
- 计划状态
- 会话状态
- 场景状态

E. 区块：学习单元清单

- 展示每个 Study Unit 的标题、页码区间、类型与是否纳入计划。
- 支持逐条编辑标题并保存。

### 3.2.3 右侧：当前计划区（PlanOverview）

A. 当前计划头部

- 当前计划标签与总量提示。

B. 计划题目与概览

- 题目展示/编辑。
- 计划 overview 展示。

C. 计划元信息

- 学习目标
- 教材
- 教师人格
- 场景使用
- 生成时间

D. 场景卡片（若已配置）

- 显示场景标题、路径、摘要、标签、焦点对象、节点数。

E. 会话动作

- 未创建会话时可直接创建学习单元会话。
- 已创建会话时显示已连接状态。

F. 排期与学习章节

- 先显示学习排期项序列。
- 每个排期项下显示只读的学习章节子项。
- 排期项支持跳转到对应教材页起点。
- 学习章节子项可点击并带页码锚点。

### 3.2.4 右侧：计划历史区（PlanHistory）

- 历史列表
  - 每条包含课程名、教材名、人格名、时间、概览。
- 切换按钮
  - 切到该历史计划并加载详情。
- 刷新按钮
  - 刷新历史快照。
- 删除按钮
  - 删除历史计划（有确认弹窗）。

## 4. 推荐使用路径

1. 进入 统一设置，确认 provider 与模型连接可用。
2. 进入 场景搭建，准备一个教学场景并保存到场景库。
3. 进入 人格色谱，配置或生成一个可用人格并保存。
4. 进入 计划生成，选择人格与场景，上传教材并生成计划。
5. 进入 章节对话，创建会话并按章节学习。
6. 进入 感官工具，按阶段调节工具开关。
7. 进入 用量审计，复盘模型成本与调用分布。

## 5. 常见问题

### 5.1 上传后没有生成计划

- 检查后端服务是否在线。
- 检查 PDF 是否有效。
- 在计划生成页查看计划轮次区，确认是否有错误与恢复信息。

### 5.2 对话无法继续

- 在章节对话页检查是否已创建会话。
- 检查当前章节/子章节是否有效。
- 使用重试按钮，并到统一设置确认对话模型配置。

### 5.3 生成人格或场景效果不理想

- 先减少输入噪声，提供更结构化关键词。
- 使用长文本提取后手工筛选卡片或节点再写回。
- 将稳定片段沉淀到卡片库/节点库以复用。

### 5.4 自动保存失败

- 在统一设置页查看状态条错误信息。
- 修正连接参数后点击重试保存。

## 6. 术语对照

- Persona：人格设定，决定教师风格与行为边界。
- Persona Slot：人格插槽，是人格的结构化组成块。
- Study Unit：从教材结构清洗后的学习单元。
- Scene：场景设定，支持层级与复用节点。
- Character Event：结构化角色表现事件，用于驱动前端角色层。
- Plan Provider：学习计划生成提供器（mock 或 litellm）。
